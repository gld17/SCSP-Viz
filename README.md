# SCSP-Viz Constellation Intelligent Computing Visualization Software V1.0

SCSP-Viz is an independent visualization delivery layer extracted from the SCSP simulation platform. The project retains configuration validation, experiment records, result export, frontend presentation, and a standardized REST API, but does not execute real simulation computation. All simulation results are generated by `scsp_viz.mock_engine`, making the project suitable for demonstrations, integration debugging, and visualization acceptance.

## Features

- Independent FastAPI service with no dependency on the SCSP_v0 source package.
- Provides a single-point simulation Mock API that returns a complete metric structure aligned with v0 `SimulationMetrics`.
- Provides a bandwidth sweep Mock API that returns knee bandwidth, knee reason, and scan rows.
- Supports experiment execution, JSONL storage for experiment records, and JSON/CSV result output.
- Supports an experiment reproduction API that generates comparison summaries from Mock data.
- Retains the configuration normalization and validation layer for frontend form display and API input checks.
- Retains scan latency visualization functions that can plot from CSV or JSON experiment result files.
- Provides a static frontend page, assets, and model registry configuration files for offline presentation.

## API Description

SCSP-Viz exposes visualization-layer interfaces according to the standardized REST API interaction specification of the SCSP simulation platform. Input fields retain the `config` dictionary format, and output structures retain the standard forms for single-point simulation, sweeps, and experiment records. In the current version, all computation results come from the Mock data layer.

- `GET /api/health`: Health check, returns `{"status": "ok"}`.
- `POST /api/simulations/single`: Single-point simulation Mock. The request body contains `config` and optional `bandwidth_gbps`, and returns `mode=config=metrics`.
- `POST /api/simulations/sweep`: Bandwidth sweep Mock. The request body contains `config` and optional `bandwidths_gbps`, and returns `mode=knee_bandwidth_gbps=knee_reason=rows`.
- `POST /api/experiments/run`: Runs a Mock experiment, outputs an experiment record and result JSON, and additionally outputs CSV in sweep mode.
- `GET /api/experiments`: Reads the JSONL experiment record list.
- `GET /api/experiments/{experiment_id}`: Reads a single experiment record.
- `POST /api/experiments/reproduce/{experiment_id}`: Regenerates Mock results from historical input configuration and returns summary differences.
- `GET /api/experiments/{experiment_id}/export`: Exports an experiment record as JSON.

## Frontend Page Description

`web/index.html` is the single-page visualization interface for SCSP-Viz. It calls `/api/simulations/*` and `/api/experiments/*` through same-origin APIs. The page reads `/assets/earth-horizon-reference.png` and `/configs/model_registry.json` for presentation and form options, but the backend does not rely on the model registry to execute real computation.
